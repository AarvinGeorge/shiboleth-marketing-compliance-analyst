# Design docs

The planning documents Adlign was built from, kept for anyone who wants to see
the thinking behind the code rather than just the code.

| Doc | What it covers |
|---|---|
| [prd.md](prd.md) | Product requirements: the mission, the problem, and the acceptance bar. WHY. |
| [spec.md](spec.md) | Technical specification: architecture, node contracts, data model, scoring, evals. WHAT. |
| [architecture.md](architecture.md) | Runtime choreography, pause/resume, frontend architecture, payload contracts. HOW. |
| [design.md](design.md) | The UI contract: tokens, aesthetic, and component rules. |

These are excerpts from a larger internal planning set, so a few in-text
references point to documents not included here. They are dated to the build
(July 2026) and were written during a three-day product trial; the product
shipped under the name Adlign. For how to run and deploy the app, see the main
[README](../../README.md).
